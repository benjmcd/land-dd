# Data Source Review: DS-022 Census TIGER/ACS

## Source identity

- Source registry ID: DS-022
- Source name: Census TIGER/ACS
- Organization: U.S. Census Bureau
- Reviewed scope: TIGERweb tract/block-group geography context only
- Excluded scope: ACS demographic variables, protected-class analytics, neighborhood desirability, market/investment/lending suitability, and residential steering
- Homepage: https://www.census.gov/data/developers/data-sets/TIGERweb-map-service.html
- API/service: https://tigerweb.geo.census.gov/arcgis/rest/services/TIGERweb/Tracts_Blocks/MapServer
- Last checked: 2026-06-11
- Review status: approved-with-restrictions

## Official authority

- Census Data API User Guide says Census applications commonly use three services: Census Data API, TIGERweb REST Services, and Geocoder REST Services.
- The same guide states TIGERweb provides Census area boundaries/shapes referenced by FIPS codes and can return boundaries by FIPS code or latitude/longitude.
- TIGERweb map-service documentation identifies the service as a GeoServices REST API.
- TIGERweb Tracts_Blocks layers list Census Tracts and Census Block Groups as polygon feature layers, with 2025 vintage descriptions and Census Bureau copyright/source text.
- Census API terms allow API use to search, display, analyze, retrieve, view, and otherwise get Census Bureau data, subject to the terms and privacy constraints.

## Rights and restrictions

| Question | Decision | Notes |
|---|---|---|
| Can we call it from the backend? | approved-with-restrictions | Bounded TIGERweb GeoServices calls only; no bulk crawling or high-volume use. |
| Can we cache source-derived records? | approved-with-restrictions | Cache only compact tract/block-group IDs, names, counts, bbox, vintage, retrieval metadata, and caveats. |
| Can we include it in reports? | approved-with-restrictions | Administrative geography context only, with Census Bureau attribution and non-endorsement notice. |
| Can we export raw data? | approved-with-restrictions | Export bounded metadata fields only; no raw geometry or demographic tables in this connector. |
| Can we use ACS demographic variables? | restricted | Not implemented. Avoid protected-class scoring, demographic desirability, residential steering, and re-identification risk. |
| Is attribution required? | yes | Include U.S. Census Bureau / TIGERweb, retrieval date, vintage, and non-endorsement notice. |
| Are there API limits? | yes | Census Data API guidance lists query limits and API-key requirements for data API use; this connector avoids ACS data API calls and keeps TIGERweb calls bounded. |

## Approved connector scope

The approved connector may:

- Query TIGERweb `Tracts_Blocks` layers for tract and block-group features intersecting a bounded EPSG:4326 bbox.
- Use `returnGeometry=false`.
- Store GEOID, name, state, county, tract, block-group code, count fields, bbox, vintage, retrieval timestamp, source URL, and caveat.
- Emit explicit source-failure evidence for network errors, malformed responses, or transfer-limit/truncation responses.

The connector must not:

- Fetch ACS demographic variables.
- Infer protected-class composition, neighborhood desirability, school quality, market value, investment quality, lending suitability, or residential steering.
- Attempt to identify any person, household, business, or establishment.
- Export raw TIGER geometry or bulk data outside the bounded review-gated connector path.

## Product caveats

- Census geography context is administrative metadata only.
- Tract and block-group boundaries are not parcel boundaries, zoning boundaries, school boundaries, market areas, or jurisdictional legal determinations.
- TIGERweb/ACS vintages can differ by layer; preserve source vintage in evidence.
- This connector deliberately excludes ACS demographic data to preserve fair-housing and protected-class safety boundaries.
