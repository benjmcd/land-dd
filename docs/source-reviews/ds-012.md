## Provenance and caveats

- Source version/date available: Each recorded instrument carries a recording date and book/page or document number. Retrieval date must be recorded. No official API for programmatic access.
- Update cadence or freshness class: Continuous — deeds are recorded as transactions close. Automated polling is not supported without confirmed API.
- Geographic coverage limits: Buncombe, Chatham, and Brunswick NC for private-MVP scope. Each county portal is independent. Not all instruments may be digitized; older records may exist only as paper or microfilm.
- Precision/scale limits: Deed records identify parties, legal description, and recording date but may reference metes-and-bounds or plat book/page that requires a licensed surveyor to interpret. Deed screening does not establish a clean title chain or confirm absence of encumbrances — professional title search is required.
- Known caveats to store with evidence: (1) Deed screening is not a title search. (2) Automated access to county deed portals requires per-county ToS review and likely a vendor API agreement. (3) Older instruments may not be digitized. (4) Legal descriptions in deeds require survey/title professional interpretation. (5) Easements, covenants, and deed restrictions recorded in deed books require manual review for completeness.
- Required attribution text: [County] Register of Deeds, retrieval date, document number and recording date.
- Fields that must not be exposed: Grantor/grantee information should not be exposed as ownership data without explicit licensed data agreement; screened for PII appropriateness.
- Source failure / no-data behavior: Record source failure when deed search is unavailable; do not silently return "no encumbrances found."
- Reviewer notes: **PRIMARY BLOCKER: No public machine-readable REST API confirmed for any MVP county (Buncombe, Chatham, Brunswick NC).** County deed portals are web-UI only, operated by county-specific vendors (iDocMarket/Laredo for Chatham; vendor portals for Buncombe and Brunswick). ToS for automated/machine access is not confirmed. A live connector cannot be implemented without: (a) per-county portal ToS reviewed and confirmed to permit automated queries, or (b) a public vendor API with confirmed license for automated use and caching. Re-review required when a confirmed API endpoint or vendor agreement is identified for one or more MVP counties.
